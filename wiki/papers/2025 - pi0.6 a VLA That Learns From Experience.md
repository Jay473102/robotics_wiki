# $\pi^{*}_{0.6}$: a VLA That Learns From Experience

## TL;DR

RECAP은 VLA를 demonstrations만으로 학습하는 데서 멈추지 않고, autonomous rollouts, sparse reward feedback, human interventions를 offline RL 형태로 반복 반영하는 training recipe다. $\pi^*_{0.6}$는 advantage conditioning을 포함한 $\pi_{0.6}$ 계열 VLA로, laundry folding, espresso making, box assembly에서 hard task throughput을 두 배 이상 높이고 failure rate를 약 절반으로 줄였다고 보고한다.

## Problem

VLA는 language prompt로 generalist robot task를 지정할 수 있지만, 실제 배포 task에서 mastery를 얻으려면 practice가 필요하다. 단순 imitation learning은 demonstration distribution 밖의 compounding error와 deployment condition 변화에 취약하다.

Real-world RL을 VLA에 적용하려면 large model 안정성, heterogeneous data, sparse/stochastic reward, human intervention 데이터 통합 문제가 생긴다.

## Core Idea

RECAP은 value function으로 action advantage를 추정하고, 그 advantage에서 나온 improvement indicator를 VLA prefix/context에 넣어 policy를 개선한다. Policy gradient를 직접 large VLA에 적용하기보다, offline RL-style value training과 advantage-conditioned behavior cloning을 결합한다.

## Method

반복 절차는 세 단계다.

1. VLA를 task에 배포해 autonomous rollout, outcome reward, 필요 시 human-gated intervention을 수집한다.
2. 지금까지 모은 데이터를 사용해 multi-task value function을 학습한다.
3. Value function이 추정한 advantage를 기반으로 improvement indicator를 만들고, 이를 condition으로 VLA를 fine-tune한다.

$\pi^*_{0.6}$는 offline RL pretraining으로 시작하고, downstream task에서 demonstrations와 on-robot data collection을 통해 추가 개선된다.

## Data / Embodiment

실험 task는 laundry folding, espresso drink making, cardboard box assembly다. 논문은 real homes, professional espresso machine, factory deployment-style box assembly를 포함한다고 설명한다.

## Evaluation

평가는 throughput과 success rate를 중심으로 한다. Baseline은 pretrained $\pi_{0.6}$, offline RL pretraining, SFT, AWR, PPO-style extraction을 포함한다.

## Results

- 논문은 diverse laundry와 espresso task에서 on-robot data를 포함한 최종 $\pi^*_{0.6}$가 throughput을 두 배 이상 높였다고 보고한다.
- Failure rate는 hard task에서 약 2배 줄었다고 설명한다.
- Espresso task는 13시간 연속 실행, novel laundry는 새 집에서 2시간 이상 interruption 없이 수행, box assembly는 factory packaging scenario에서 수행했다고 보고한다.
- Policy extraction 비교에서 RECAP은 AWR/PPO-style extraction보다 laundry throughput이 높다고 보고한다.

## Limitations

- RECAP은 real-world rollout과 human intervention을 요구하므로 data collection 비용과 safety gating이 중요하다.
- Value function error가 advantage label 품질을 좌우한다.
- 논문은 task set이 laundry, espresso, box assembly 중심이므로 mobile manipulation, high-force contact, safety-critical deployment로의 일반화는 추가 검증이 필요하다.
